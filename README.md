# Coinbase Search API
[![Run in Postman](https://run.pstmn.io/button.svg)](https://app.getpostman.com/run-collection/808fe1b2aca7edde8574)

This API exposes 3 endpoints to list Currencies, Trading Pairs, and Historical Rates. Each endpoint supports filtering, sorting, and cursor pagination. Search results are cached using low-level Rails caching, HTTP caching is enabled for all external requests, and of course database queries used multiple times per action are cached. Postgresql full text search is implemented via the `pg_search` gem. And API documentation, generated by the `rswag` gem, can be found [here](https://coinbase-search-api.herokuapp.com/api-docs).

## Usage
The API is secured using a very basic Bearer token scheme, with only one token generated for simplicity. A production version of this API would likely have Users with tokens stored for lookup in place of the hard-coded token currently used. 

### Postman Collection
A Postman collection for the API is available [here](https://www.getpostman.com/collections/808fe1b2aca7edde8574).

### Authorization
The token for authorizing is `73e2f213f7d875e9e62798b61d3c275ec63f2efe`. Attach it via the standard HTTP Authorization header with a Bearer scheme:
`Authorization: Bearer 73e2f213f7d875e9e62798b61d3c275ec63f2efe`

### Content Type
This API produces and consumes JSON, and as such requires a JSON Accept header to prevent an error:
`Accept: application/json`

## Endpoints 
While this document shows basic endpoint usage, more thorough documentation for all endpoints can be found [here](https://coinbase-search-api.herokuapp.com/api-docs). Be sure to click the `Authorize` button, enter the API token in Bearer scheme, and  select the production server from the Server dropdown.
